    • Instructor Usage

    1. Connect to XData from course management system; if course management system is not used, and XData is connected directly, open http://*root-url*/XDataWeb, and login with the instructor username and password given by the XData system admin. Please check Administrator UsageAdministrator Usage for details.
    2. You will be forwarded to Course Home page if you login directly to XData system. It will show you the list of courses for which you are registered. Click on the view button to view the course details. If you are instructor to the course and it is a new course, please proceed with the following steps to create assignment.
        3. New Database Connection:
      1. This feature allows you to create a connection to any database for executing student queries [We are working on providing support to multiple databases like Oracle, MySQL,etc.,]. The default database used is postgreSQL.
      2. The database user name and password will be the the user name and password for the schema which the instructor will use for generating datasets for the questions in the assignment.
      3. The test user name and password will be the user name and password for the schema in which the student queries will run against the correct query
        Upload Schema:
      1. This feature allows you to specify a SQL script containing the schema. Click on Choose File, browse to the file and click on Upload button. Sample schema can be found here
      2. The uploaded file can be tested for correctness when "Test and Upload" button is clicked.
      3. If the file is uploaded without testing, it can be tested later by using "Test" button in the existing schema list.
      4. For testing the schema file, database connection needs to be selected. This testing will use the database username and database password that was provided when new database connection was created.
        Upload Sample Data:
      1. This feature allows you to specify a SQL script containing the initial test data. Choose an appropriate Schema Name from the drop down list, it will show option to upload a new schema file and show a list of data files existing for that schema.
      2. click on Choose File and browse to it, and finally click on the Upload button to submit it. Sample Data can be found here.
      3. The uploaded file can be tested for correctness when "Test and Upload" button is clicked.
      4. If the file is uploaded without testing, it can be tested later by using "Test" button in the existing schema list.
      5. For testing the schema file, database connection needs to be selected. This testing will use the database username and database password that was provided when new database connection was created.
        Assignment Assignment allows you to create new one or view existing one.
        1. New Assignment:
        1. This feature that allows you to setup a new assignment for the students.
        2. You can enter the name and description of the assignment, along with the default schema to be used for this assignment. You can specify the start and end time for the assignment for imposing a deadline.
        3. You have to select the database connection for the assignment. This connection will be used for testing the student query. We use the test database and test password provided during "New database connection". After supplying all settings click on Submit button.
        2. View Assignments:
        1. This feature allows you to view all the assignments created by the you. On click of the desired assignment id link, you can Edit Questions, Grade Questions, View the status of question, View Results and scores after evaluation of questions
        2. The assignment details also provides option to edit the assignment details like start date, end date, name, description, etc.,
        3. There is a button "Copy Link to Clipboard" which onclick copies the url into clipboard. This URL can be used to connect to that assignment in XData system directly from any external learning tool like Moodle.
        4. The options given to the instructor are as follows :
            View/Edit question:
          1. You can view the questions that are existing for the assignment. If it is a new assignment, you can add the new question text and SQL answers.
          2. Multiple SQL answers: If more than one answer is to be given for a question, you can click "Add Another Query" and enter the other answer. In case of multiple answers, can also specify whether the student's answer must be evaluated against all answers or any one answer.
          3. Marks scroller:The scores to be awarded for the question is specified using this scroll bar.
            If there are multiple answers to one question, the instructor may choose to award different marks for each answer.
          4. Multiple Schema: If a question belongs to different schema other than the default schema specified during assignment creation, the drop box with list of schema can be used to select a different schema.
          5. Once all these are done, click on update and the assignment will be updated with the queries.
          6. Generate Data: Once the question and appropriate answers are added, you can generate the data sets for each answer by using "Generate Data" link
          7. Wait till "show generated data" link appears and click on the link to view the data sets generated for the SQL answer.
          8. If any error occurs in data generation, please check the log files for details. Use "Edit" link to edit the question text or SQL answer. Once the SQL query is edited, the generated data will be lost and repeat step 6 again.
            Grade Question:
          1. The student answers are graded using this link
          2. Evaluate Question: Onclick of evaluate question link, the system gets all students answers for that question and evaluates it against the instructor answer and marks are assigned to the students answer accordingly.
          3. Status Of Question: This link is used after evaluation of a question to see the students who had answered correctly, their scores, if student had answered incorrectly, it shows link to see the mark details provided by the Partial Marker.
          4. Partial Marking: XData system provides partial marks to the student answer if the student query validation fails against the instructor query.
            The tool gives you the ability to override the marks allotted through the automatic assessment. The student query and the instructor query is displayed in the partial marking details page. This helps you to verify and correct the marks. If you wish to award more / less marks than the generated mark, this page provides a text box to override the marks assigned.
            View Results:
          1. After the evaluation, this link gives the result per question like how many answered correctly and how many answered incorrectly and the correct answer.
            View Scores:
          1. After evaluation of all questions in the assignment, View Scores gives the total score of each student for that assignment. The upgrade link helps in upgrading the student marks to the External Learning Tool.
