XDATA SYSTEMS AND USERS

The XData grading tool which can be used by instructors and TAs for grading SQL query assignments automatically. Provided with one or more correct queries for an SQL assignment, the tool uses the XData system to automatically generate datasets that are designed specifically to catch common errors. The grading is then done by comparing the result of student queries with that of the correct queries against these generated datasets as well as additional datasets optionally provided by the graders.

The users of the system will be the TA's and instructors of the database courses and students who want to learn SQL

PROCESS FLOW FOR ADMIN

Xdata systems has an administrator mode that helps in creating the users, courses, etc., One can connect to the XData server using the URL http://<host-url>/XDataWeb, and try to login with the username as 'admin' and the password set during installation. The user will see the welcome screen with administrator menu.

The administrator should follow the steps given below:

1. Create course for Xdata systems

2.Create the users with login information and emails using 'Create user' link.

3.Assign roles for a user providing user's login id using the 'Assign Roles' link. First admin will enter login id of the user and then select a course and a role for that user.

4.The admin can edit the details of an existing course or delete courses using the 'View Courses' link. Similarly, the admin can edit user information or delete the user using 'View Users' link.

5. The admin has to set-up LMS details – the instance URL of LMS accessing Xdata, Consumer Key and Secret Key. Please see the LMS set up to know about these.

PROCESS FLOW FOR INSTRUCTOR

If the instructor's course is integrated with a Learning Management System, the instructor should add the URL : 'http://<host-url>/XdataWeb/tool.jsp' as external URL in LMS assignment. First an assignment needs to be created in Learning Management System and this URL needs to be used. Read the instructions at PROCESS FLOW FOR LMS for instruction on how to set up an assignment link.

Instructor should login to XData server using the URL http://<host-url>/XdataWeb. Instructor can get the login details from the administrator.

On login,course list page will show the list of courses for which the user is registered. On click of 'View', it will show the list of assignments available for that course.

Once the instructor menu is shown, the following steps needs to be followed:

DATABASE CONNECTION

This link is used for maintaining connections to databases. details to run the instructor query on data sets and to evaluate student query.. The details to be given are :

a. Database Connection Name: A user-defined connection name for identifying the connection.

b. Database Name: The database name created for the assignment.

c. Database Type: The type of database . (We currently support only PostgreSQL but are working on support for multiple databases like Oracle, MySQL and Microsoft SQL server.)

d. JDBC URL: The jdbc URL for the database in the format <hostname:port>.

e. Database User name and Password: A database username and password which will be used for generating datasets for the SQL queries in the assignments.

f. Test User name and Password: A test username and password which will be used to evaluate student queries.

After all connection details are entered, the connection is tested for correctness of data and database. The details are added in Xdata system.

UPLOAD SCHEMA

This link is used to upload an SQL script containing DDL statements to create database tables. The SQL queries for the assignment will be based on the tables given in the schema. First the 'Choose File' button is used and file is selected. Name of the schema can be given for identification in Xdata Systems. If Name is not given, filename will be used as the schema name.

  1. 'Test and Upload' button is used to test the SQL file on a database connection for syntax errors and if no errors are there, file is uploaded.

  2. 'Upload' button uploads the file without testing it.

  3. Any schema can be tested against various connection later by using 'Test' button in the schema list.


UPLOAD SAMPLE DATA

    The 'Sample Data' link is used to upload SQL script that contains sample data. First the schema needs to be selected for which the data is to be uploaded. If already data file exists, they are listed for testing, viewing and deletion.

    If no file exists, the Sample data file which is to be used for data generation is uploaded first. Then the Sample datasets for evaluating the student query can be uploaded.

  1. 'Test and Upload' button is used to test the SQL file on a database connection for syntax errors and if no errors are there, file is uploaded.

  2. 'Upload' button uploads the file without testing it.

  3. Any data file can be tested against various connection later by using 'Test' button in the schema list.

CREATE ASSIGNMENT

Assignment part has two links. One to create a new one and other to view existing one. For new assignment, user inputs the following:

Assignment Name,

Assignment Description,

Assignment Mode,

Default schema to be used for this assignment.

Default datasets for evaluation of assignment (Additional datasets provided by instructor using “upload sample data” UI on which student queries will be tested.)

Start date and time and

End date and time for the assignment for imposing a deadline.


    Assignment Modes

  1. Interactive: “Interative” check box, if selected, creates interactive assignments that the students use for learning. If assignment is marked interactive, then the student query will be evaluated against instructor's answer as soon as the student submits the answer. The student will be able to see the correct answer given by the instructor and the datasets in which his query failed if its incorrect and thus interactively learn from his mistakes.

  2. Grading: If "Interactive" is not checked, then when student submits the query it will be checked against basic test case and student answers will be saved. The instructor has to manually start the evaluation after the due date of assignment or after all students submit their answers.

  3. The instructor has to select the database connection for the assignment. This connection will be used for testing the student query. Xdata systems uses the test database and test password provided during "New database connection" for running evaluation. After all settings are given, the assignment creation is triggered.


VIEW ASSIGNMENTS

This is an important and most used UI to view all assignment created by the instructor for that course. On click of the desired assignment name or id, Assignment Details is shown in which the following information are there.


  1. The assignment details also provides option to edit the assignment details like start date, end date, name, description, etc.,

  2. The assignment can be deleted (which internally deletes all questions, student answers, marks details and anything related to that assignment) using Edit assignment UI

  3. There is a button "Copy Link to Clipboard" which on click copies the url into clipboard(This feature works in Chrome and Safari browser. For firefox browser please enable Flash support. If you are not interested in enabling flash, Please copy paste the URL). This URL can be used to connect to that assignment in XData system directly from any external learning tool like Moodle.
    Also note the consumer key and secret key values in the message were given during installation. These values will be used when connecting to Xdata from Course management system.

  4. Links for Add/View Questions, Result summary by Question and View scores

  1. Add/View Questions

    On click of Add/View Questions, list of questions in the exiting assignment are listed below the assignment description box. The “Evaluate Assignment” link is used to evaluate all questions in the assignment.

    If it is a new assignment “Add Question” button is displayed which will take the user to the “Add Questions” screen. The instructor has to type in the question text and the SQL answer for the question text. Each question in the assignment has the following links.

    View/Edit question :

    This link shows the questions that are existing for the assignment with options for editing each question,evaluating each question, generation of dataset, evaluation result, delete question,etc,.

    1. On click of Edit link, the question can be edited – partial mark parameters, marks allotted for the question, default datasets for evaluation of that particular question, question text, SQL answer, etc., can be edited and changes can be saved. XData provides some additional features that helps in grading the assignment better. They are as follows.

    Multiple SQL answers:

    While adding a question and an answer to the assignment, if more than one SQL answer is to be given for a question, "Add Another Query" button can be used and other answer can be added and saved. In case of multiple answers, a radio button appears below the text area where instructor can specify whether the student's answer must be evaluated against all answers or any one answer.

Marks:

The instructor can use the mark slider above the question to set the maximum marks awarded for the question. There is another mark slider above each and every SQL answer box, this is used to set the maximum marks for every answer. For one exactly correct answer, instructor can award 100 marks and for another answer instructor can award 75 or 50 marks for other answer.

Partial Marking Parameters:

There is a “View / Set Partial Marking Parameters” link above each answer box. This is used for setting weightage for partial marks. The partial marking feature in the grading tool assigns marks to student queries based on a heuristic of how close the student is to to the instructor's query. It ensures to ensure a standard and consistent evaluation scheme across all students.

Partial marking is done only if the student query fails one or more datasets during evaluation. For these student submissions the instructor can click on the 'Details' link in “Evaluation Status” page to see how the weightage of partial marks were awarded. Based on this, the student query is evaluated and partial marks are awarded if the parameters match.

Match All / Any:

1. Once “Add another Query” button is clicked and second query is added, the radio box for “Match All/Any” is shown. If “Match All” is selected, while evaluation – output of the query on all the datasets needs to be matching the instructor output. If ”Match Any” is selected, while evaluation – output of the query on any one answer's datasets needs to be matching the instructor output.

2. On click of “Evaluate Question” link, all the student answers for that question will be evaluated and the “Evaluation result” link is shown.

3. “Generate Dataset” link is used to generate the datasets for given SQL answer. The SQL answer is parsed and datasets are generated for each part of the query. Evaluation is done based on the student and instructor query output on these datasets.

4. “Show Dataset” link is used to show the datasets generated. This UI shows the data generated for the tables used in the SQL answer as well as data generated for the reerence tables and the instructor output on that dataset.

5. “Delete” link is used to delete the question along with the SQL answers and generated datasets.


2. Result Summary by Question

      This link shows the results summary. It shows a table that contains list of SQL queries in the assigment along with question text, number of students who have answered it correctly and number of students who have answered in wrongly.

3. View scores

    This link shows the UI with a table that holds all the student names, their e-mail id, total assignment score, and a “Details” link. If the login is from Learning management system like moodle, blackboard, etc., there will be an “Upload” link above the table as well as in each row of the table.

    The “Upload” link above the table is used to upload all the students marks to the LMS system.

    “Upload” link in each row is used to upload the selected student mark to the LMS system.

    On click of the “Details” link, that shows a table that has “Student Answer” , “Status” ,”Marks” , “Mark details” and “Error Details” link. If the student answer is correct, then these links will not be shown.

Mark Details shows the weightage of the marks awarded, an option to change the obtained marks, student answer and instructor answer for the query.

Error Details shows the datasets in which the student query failed, student output on that dataset and expected output on that dataset. It also shows the datasets on click of a link.


PROCESS FLOW FOR LMS


The instructions provided below are specific to setting up the tool in Moodle system. However, they can be easily extended to other learning management systems as well (provided the learning management system supports the LTI standard).

The XData grading tool can be added in the Learning Management System as an external tool. In Moodle, the instructor can use the 'Turn Editing On' button to add an activity or resource and select 'External Tool' from the list of options displayed.

The instructor needs to provide a name for the external tool. This name will be displayed on the course page as shown below. set the Launch URL as 'http://<host-url>/XdataWeb/tool.jsp'

After adding the URL, the instructor needs to use the 'Show More...' link and add the 'Consumer Auth Key' and 'Secret Key' for using XData. These keys are setup during installation and are used for the purpose of OAuth authorization.

The instructor can connect to XData and create an assignment using this link. Once the assignment is created, the instructor can get an assignment URL using 'Copy External Toollink' given in the 'Assignment Details'. The instructor can now edit the assignment and set the external tool link to this URL. The assignment link from the learning management system will take the student directly to the assignment set by the instructor.

Once the assignment evaluation is over, the instructor can go to 'View Assignment -> View Scores' page on the grading tool, check the marks of each student and click the upload button. This will upload the student marks to the learning management system.



    LIMITATIONS

Although Xdata systems is tested to work on many test case queries, few cases are not handled.


1. Queries with Case statements

2. Queries with lateral subqueries

3. Subqueries within subquery (Nested subqueries)

4. Set queries with aggregations

5. With As within With As queries

6. Queries with Extract expression for column name that holds date value are approximated.

7. Queries with Extract expression for time and timestamp values on column name are not handled.

7. EXISTS / NOT EXISTS without any subquery conditions
